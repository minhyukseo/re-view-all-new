# 영상뷰어 모드(Video Viewer Mode) 분석

## 2. Analysis (분석) - [본질가] 주도
**본질가**: 각 플랫폼의 HTML 구조를 분석해 보니, 단순 Regex로는 한계가 명확하오. 메타 태그(`og:video`)와 실제 미디어 노드를 정밀 타격해야 하오. Worker의 `Edge Runtime` 제약을 고려한 가벼운 파싱 알고리즘이 필수적이오.

**실천가**: 분석 끝남. 유튜브는 `iframe` 코드만 따오고, 일반 사이트는 `.mp4` 링크만 추출하면 됨. `fetch`로 HTML 긁어온 다음에 문자열 인덱싱으로 소스 뽑아낼 거임. 

**회의론자**: 하, 그게 말처럼 쉽나요? 다이나믹 렌더링 사이트는 정적 파싱으로 안 뚫립니다. 그리고 Cloudflare Worker IP는 이미 많은 사이트에서 블랙리스트에 올렸을 텐데 말입니다. 

**사용자 대변인**: 아... 그런 세세한 건 모르겠고, 아무 사이트 링크나 넣어도 마법처럼 짠! 하고 영상이 나왔으면 좋겠어요. 끊기면 화낼 거예요!

**통합자**: 분석 결과, (1) 정적 HTML 기반 소스 추출, (2) 주요 플랫폼 전용 어댑터, (3) 실패 시 일반 `iframe` 임베드 시도라는 3단계 전략이 도출되었습니다.
