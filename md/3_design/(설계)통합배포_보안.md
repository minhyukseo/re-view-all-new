# 배포 파이프라인 통합 및 보안 설계 (Design)

## 1. 보안성 강화 (토큰 탈취 방지)
- **기존 설계의 치명적 오류**: Git을 통해 추적될 수 있는 `deploy.sh`에 API Token(C6gWN-...) 정적 할당.
- **개선 설계**: 
  - Token을 소스코드와 분리하여 `.env`로 격리.
  - 리포지토리에는 구성 양식인 `.env.example`만 업로드되도록 설계.
  - 실행 전 `CLOUDFLARE_API_TOKEN` 존재 유무를 검사하는 'Fail-fast' Guard 적용.

## 2. 통합 파이프라인 트리거 구축
- **기존 설계**: 사용자가 Web, Backend(API), Edge를 각기 별개 스크립트와 패키지로 다루어 혼선 야기. 
- **개선 설계**: 
  - 단일 스크립트 `deploy_all.sh`에서 상호 의존성을 고려해 **Server(API) ➔ Edge Worker ➔ Web(Front)** 순서로 순차적 빌드 및 배포 처리.
  - 최상위 `package.json`에 `deploy:all` 명령어 주입을 통해 인터페이스 단일화(`npm run deploy:all`).

## 3. 데드 코드 삭제
- 실제 유효하지 않은 `Scripts: { "deploy:cloudflare" }` 레거시 및 보안 취약 `deploy.sh`를 파기.
