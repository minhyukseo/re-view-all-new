# 영상뷰어 모드(Video Viewer Mode) 설계

## 3. Design (설계) - [본질가] vs [사용자 대변인]
**본질가**: UI는 완벽한 어둠을 지향해야 하오. 사용자가 입력 필드에 집중하지 않도록 영상 재생 시에는 오직 영상과 최소한의 컨트롤러만 남아야 하오. API 응답은 `{ videoUrl: string, poster: string, title: string, platform: string }` 포맷으로 통일하겠소.

**사용자 대변인**: 아까부터 어둠 어둠 하시는데, 너무 삭막해요! 영상 주변에 은은한 오로라 같은 배경 불빛(Ambient Light) 좀 넣어주세요. 그리고 재생 목록도 옆에 슬쩍 보였으면 좋겠고요!

**회의론자**: 오로라라니, 렌더링 부하만 늘리시는군요. 저해상도 영상일 때 UI가 예뻐봤자 자괴감만 들 텐데요? 최소한 자동 재생 차단 정책(Autoplay Policy)에 대응할 버튼 하나는 제대로 만드시길 바랍니다.

**실천가**: 대충 합의됨. 백엔드는 `/api/video-extract`에서 정규식으로 소스 따오고, 프론트는 `Next.js`의 `/video` 페이지에서 `VideoPlayer` 컴포넌트 호출할 거임. 오로란지 뭔지는 배경에 `blur` 좀 먹여서 처리함.

**통합자**: 설계안 확정입니다. (1) 시네마틱 다크 테마 UI, (2) 앰비언트 라이트 효과, (3) 통합 응답 규격 API를 기반으로 구현 단계로 넘어갑니다.
