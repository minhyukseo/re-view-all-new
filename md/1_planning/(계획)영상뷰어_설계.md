# 영상뷰어 모드(Video Viewer Mode) 기획/계획

## 1. Planning (기획/계획) - [본질가] 주도
**본질가**: 영상뷰어의 본질은 '노이즈 제거'와 '몰입'에 있소. 복잡한 커뮤니티 UI를 걷어내고 순수하게 영상 그 자체에만 집중할 수 있는 독립적인 샌드박스 환경을 구축해야 하오. 단순한 임베드가 아니라 소스를 직접 추출하여 제어권을 가져오는 것이 핵심이오.

**실천가**: 아 말이 너무 길음. 그냥 `/api/video` 만들어서 메타데이터 긁고, `ReactPlayer` 때려박으면 끝임. 복잡하게 생각하지 말고 빨리 구현하고 깃푸시나 하자고.

**회의론자**: 아이고, 퍽이나 쉽겠습니다? 유튜브나 커뮤니티들이 바보도 아니고 크롤링 다 막아놓을 텐데, CORS 에러랑 403 Forbidden 맛 좀 보시겠군요. 보안관 입장에서 이건 API 오용의 소지가 다분해 보입니다만?

**사용자 대변인**: 너무 어려워요! 그냥 유튜브 링크 넣으면 꽉 찬 화면으로 예쁘게 나오기만 하면 돼요. 글래스모피즘인가 그거 꼭 넣어주세요! ㅠㅠ

**통합자**: 자, 종합해 보겠습니다. 영상뷰어 모드는 (1) 백엔드에서 영상 소스를 정밀 추출하고, (2) 프론트엔드에서 시네마틱한 전용 플레이어를 제공하는 '집중형 뷰어'입니다.
