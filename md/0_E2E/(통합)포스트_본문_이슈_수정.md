# 통합 보고서: 포스트 상세 페이지 이슈 해결 (Llama 3.1 8B 연동 후속)

본 보고서는 `https://re-view-all-web.seomh81.workers.dev/post/4147` 등에서 발생한 본문 텍스트 깨짐(`&nbsp;` 노출) 및 이미지 엑박 이슈에 대한 분석 및 해결 과정을 기술합니다.

## 1. 개요
- **현상 1**: 본문 및 제목 내 HTML 엔티티(`&nbsp;`, `&amp;` 등)가 디코딩되지 않고 텍스트로 그대로 노출됨.
- **현상 2**: 개드립(dogdrip.net) 등 특정 커뮤니티의 이미지가 리퍼러 정책으로 인해 엑박으로 표시됨.

## 2. 분석 결과
- **원인 1**: 추출기(Extractor)가 저장한 텍스트에 HTML 엔티티가 포함되어 있으나, React 컴포넌트에서 이를 단순 String으로 렌더링하면서 발생.
- **원인 2**: 기존 이미지 프록시 로직이 디시인사이드(dcinside)에만 한정되어 있어, 타 사이트 이미지는 직접 호출을 시도하다 차단됨.

## 3. 해결 조치
- **텍스트 디코딩**: `web/lib/decode.ts` 유틸리티를 생성하여 주요 HTML 엔티티를 안전하게 디코딩하는 로직 구현.
- **이미지 프록시 확장**: `web/lib/media.ts`의 프록시 대상을 `dogdrip.net`, `fmkorea.com`, `theqoo.net` 등으로 확대.
- **방어적 렌더링**: `web/app/post/[id]/page.tsx`의 `<img>` 태그에 `referrerPolicy="no-referrer"`를 추가하여 프록시 미작동 시에도 리퍼러를 숨기도록 조치.

## 4. 변경 파일 목록
1. `web/lib/decode.ts` (신규): 엔티티 디코딩 로직제공
2. `web/lib/media.ts`: 프록시 호스트 추가 및 함수 범용화
3. `web/app/post/[id]/page.tsx`: 상세 페이지 렌더링 로직 개선
4. `web/components/PostCard.tsx`: 목록 카드 제목 디코딩 추가

## 5. 실행 결과
- 본문 내 `&nbsp;`가 공백으로 정상 전환됨.
- 개드립 등 외부 이미지가 프록시를 통해 정상적으로 로드됨.
