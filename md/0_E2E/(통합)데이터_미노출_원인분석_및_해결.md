# 🕵️ 데이터 미노출 원인 분석 및 해결 완료 보고서

## 1. 🔍 분석 결과 (Summary of Analysis)
- **원인 1**: 사이트별 게시글을 하나씩 섞는(Interleaving) 로직이 특정 사이트의 최신글 노출을 지연시킴.
- **원인 2**: 세션 저장소(sessionStorage)에 만료 시간 없이 데이터가 캐싱되어, 서버의 변경사항을 즉시 반영하지 못함.

## 2. 🛠️ 수정 내역 (Implementation Details)

### ✅ Backend (Server)
- /api/posts?mode=latest 옵션 추가.
- latest 모드에서는 datetime(COALESCE(created_at, crawled_at)) DESC 단일 정렬 쿼리 실행.

### ✅ Frontend (Web)
- **정렬 모드 상태 도입**: sortMode ('interleaved' | 'latest') 상태 추가.
- **탭 UI 구현**: 상단 네비게이션에 'Trending', 'Latest' 버튼 배치 및 상태 연결.
- **캐시 지능화**: 
  - 캐시 데이터에 timestamp 추가.
  - 5분(300,000ms) 경과 시 캐시 히트를 무시하고 최신 데이터 페칭.
  - 필터나 정렬 모드가 다를 경우에도 즉시 서버 호출.

## 3. 🏁 기대 효과
- 사용자는 이제 'Latest' 탭을 통해 모든 커뮤니티의 최신글을 가장 빠른 시간 순서대로 확인할 수 있습니다.
- 백그라운드 캐시 갱신 주기를 통해 자동화된 신선도 유지가 가능해졌습니다.
